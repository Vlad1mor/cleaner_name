# Russian Name Cleaner

## Очищает имена от лишних символов (пунктуация, пробелы, латиница, цифры) и загружает в PostgreSQL.

## Использовал инструменты, методы:

- Python: среда выполнения всего проекта

- PostgreSQL: целевая база данных для хранения очищенных имён

- Git: контроль версий, отслеживание изменений кода

- Poetry (или pip): управление зависимостями

- pandas: чтение файлов (CSV, Excel, JSON, TXT) и работа с DataFrame

- re (встроенный модуль): регулярные выражения для удаления всего, кроме русских букв

- sqlalchemy: создание подключения к PostgreSQL и загрузка DataFrame через to_sql

- psycopg2-binary: драйвер PostgreSQL для SQLAlchemy

- openpyxl: чтение Excel-файлов (.xlsx, .xls)

- python-dotenv: загрузка переменных окружения из файла .env (логин, пароль, имя БД, хост)

- sys (встроенный): чтение аргументов командной строки (имя файла)

- os (встроенный): работа с путями и проверка существования файлов

- pgAdmin / psql: визуальное или консольное администрирование PostgreSQL и проверка данных
Очищает имена от лишних символов (пунктуация, пробелы, латиница, цифры) и загружает в PostgreSQL.

## Форматы входных файлов

- .txt – каждая строка становится отдельной записью
- .csv – обрабатываются все текстовые колонки
- .xlsx / .xls – обрабатываются все текстовые колонки
- .json – обрабатываются все строковые поля


## Установка

```bash
poetry install
```

Настроить подключение к PostgreSQL в файле `.env`:

```
DB_HOST=localhost
DB_PORT=5432
DB_NAME=your_database
DB_USER=your_user
DB_PASSWORD=your_password
```

## Запуск

```bash
python src/main.py names.txt
```

Вместо `names.txt` укажите имя вашего файла (лежит в папке `data`).

## Что делает скрипт

1. Читает файл из `data/`
2. Удаляет всё, кроме русских букв (`а-яА-ЯёЁ`)
3. Отбрасывает строки, ставшие пустыми
4. Добавляет очищенные записи в таблицу `cleaned_names`

## Проверка результата в PostgreSQL

```sql
SELECT * FROM cleaned_names LIMIT 10;
```
